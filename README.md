# Локальная RAG-система на базе Saiga3 (Llama-3-8B) с кастомным AI-агентом

Реализация архитектуры **RAG (Retrieval-Augmented Generation)** для локального анализа PDF-документов без отправки данных на внешние серверы и сторонние API. 

В качестве ядра системы используется AI-ассистент **«Арсений»** (промпт имитирует поведение строгого ML-инженера).

## 🛠️ Стек технологий
* **LLM:** `Saiga_Llama3_8b` в квантованном формате GGUF (`Q4_K`), развернутая через `llama-cpp-python`.
* **Эмбеддинги:** Мультиязычная модель `paraphrase-multilingual-MiniLM-L12-v2`.
* **Векторный индекс:** `FAISS` (локальное сохранение индекса).
* **Оркестрация цепочки:** `LangChain`.
* **Парсинг PDF:** `PyMuPDFLoader`.

## 🚀 Как это устроена внутри

### 1. Двухэтапный поиск контекста (Multi-Query + Reranking)
* **Multi-Query Retriever:** Исходный запрос пользователя перефразируется моделью 3 раза с использованием профильной терминологии, чтобы расширить область поиска в векторной базе.
* **Cross-Encoder Reranker:** На базе компактной модели `TinyBERT` выполняется семантическое переранжирование найденных чанков. В модель передается только `top_n=3` наиболее информативных фрагментов, чтобы отсечь лишний шум.

### 2. Защита от галлюцинаций
В системном промпте агента жестко прописаны ограничения: оперировать исключительно фактами из предоставленного текста. При отсутствии прямой информации в контексте агент выдает фиксированный ответ: *"Коллега, нет данных по данному вопросу."*

### 3. Модуль безопасности (Guardrails)
Интегрирована простая функция `safety_check` для фильтрации опасных запросов на этапе входящего потока (до вызова тяжелой LLM). При обнаружении стоп-слов выполнение блокируется.

## 📊 Результаты
Блокнот содержит примеры работы системы в среде Google Colab: успешный сбор ответов по базе знаний (на примере работы FAISS) и корректную отработку защиты при вводе деструктивных запросов.
